Review of current HTML5 bugs as of 9 June 2015

WARNING: This is a rough review from one engineer on the Microsoft Edge team to give an initial indication of prioritisation. It does not represent a precise "Microsoft position". There may be some mistakes. There are almost certainly some items where we will change our position after more review. The goal is a first pass to understand the nature of the bugs currently open and their relative magnitude.

Bug Type

BugThis is a bug in the spec.
EditorialThis is an editorial change that doesn't change implementations.
Minor newThis is a "bug sized" new feature / small extension to existing feature.
FeatureThis is a feature request.

Priority

For bugs with Bug Type = "Bug", the following priorities are assigned:

P0High priority - affecting live sites today.
P1Medium priority - interop issue affecting different browsers and likely to be impacting web developers.
P2Low priority - should probably be fixed but not a priority for Microsoft.
P3Recommend resolve with no further action.

We consider the types of bugs designated P0 and P1 bugs to be the kind of maintenance activity we envisage working on in the short term.

For other bugs, the priority value is simply a guess at the relative priority.
